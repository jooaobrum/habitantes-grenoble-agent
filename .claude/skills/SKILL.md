# Skills

Skills are playbooks you load in Copilot when you need them.
Load a skill by referencing it in chat: `#file:skills/<name>/SKILL.md`

## When to use which skill

| I need to... | Use |
|---|---|
| Start a new project from the ideation brief | `project-bootstrap` |
| Write or update spec / design / tasks | `spec-driven-ds` |
| Review or design the architecture | `agent-system-architect` |
| Simplify something that got too complex | `architecture-simplify` |
| Clarify the domain language and boundaries | `domain-discovery-ddd` |
| Design the UI | `frontend-blueprint` |
| Define the UI ↔ API contract | `frontend-api-bridge` |
| Improve a system prompt | `prompt-editor` |
| Set up config (YAML + Pydantic) | `config-driven-settings` |
| Write or update docs | `docs-writer` |
| Set up Docker | `docker-builder` |
| Review code quality | `coding-guidelines` · `python-dev` |
| Apply the right design pattern for new code | `python-patterns` |
| Create a new reviewer subagent | `subagent-creator` |
| Enforce commit message standards | `github-commit-assistant` |
| Guide for kanban board | `github-kanban-assistant` |
